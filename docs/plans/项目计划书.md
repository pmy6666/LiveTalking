# LiveTalking 对话界面 SSVEP 选项选择项目计划书

## 1. 项目背景

当前 LiveTalking 已经具备数字人角色选择、WebRTC 音视频连接、自由聊天、文本播报和三选一的选项对话能力。现有选项对话位于前端 `web/dashboard.html` 的“选项对话”标签页中，由 `web/client.js` 调用后端 `/choice/init`、`/choice/select`、`/choice/reset` 等接口驱动。

本项目计划在现有“选项对话”界面中增加 SSVEP 选择支持，使每个可选项以不同频率闪烁，为后续脑机接口识别用户注视目标、自动触发选项选择提供前端刺激基础。

## 2. 项目目标

1. 在现有选项对话按钮基础上增加 SSVEP 闪烁模式。
2. 每个选项绑定稳定、互相区分的闪烁频率。
3. 保持现有鼠标点击选择能力不受影响。
4. 提供可开关的 SSVEP 模式，方便普通交互和实验交互切换。
5. 预留后续 BCI 识别结果接入入口，使外部识别模块可通过接口或前端事件触发 `choice/select`。

## 3. 现状分析

### 3.1 前端现状

当前前端主要文件：

- `web/dashboard.html`：页面结构、选项对话 tab、样式定义。
- `web/client.js`：角色加载、WebRTC 建连、选项对话初始化和选择逻辑。

现有选项渲染逻辑在 `renderChoiceState(payload)` 中完成：

- 接收 `payload.current.choices`
- 清空 `#choice-options`
- 为每个 choice 创建 `.choice-btn`
- 点击 `.choice-btn` 后调用 `requestChoiceSelect(choiceId)`

这说明 SSVEP 前端可以优先在 `.choice-btn` 的渲染、样式和动画调度层完成，暂不需要推翻后端选择树结构。

### 3.2 后端现状

后端已提供：

- `/choice/init`：初始化选项对话。
- `/choice/select`：选择当前节点下的某个选项。
- `/choice/state`：获取当前状态。
- `/choice/reset`：重置选项对话。

`choice/orchestrator.py` 中的 `_build_choices()` 当前返回：

```json
{
  "choice_id": "...",
  "choice_text": "...",
  "child_node_id": "..."
}
```

第一阶段可以在前端按选项序号自动分配频率；第二阶段可将 `ssvep_frequency`、`ssvep_phase`、`ssvep_color` 等元数据加入 choice tree 配置，使每个业务选项拥有可配置刺激参数。

## 4. 功能范围

### 4.1 第一阶段：前端 SSVEP 闪烁原型

实现内容：

1. 在“选项对话”区域增加 SSVEP 开关。
2. 渲染选项按钮时为每个选项分配频率。
3. 使用 `requestAnimationFrame` 控制闪烁，避免大量 CSS 动画难以统一暂停和重置。
4. 在按钮上展示频率标签，例如 `10Hz`、`7.5Hz`、`6Hz`。
5. 在选择状态刷新、停止连接、重新开始时正确停止或重置闪烁。

建议默认三选一频率：

| 选项 | 频率 | 说明 |
|---|---:|---|
| 选项 1 | 10Hz | 常见 SSVEP 频率，60Hz 屏幕下可稳定呈现 |
| 选项 2 | 7.5Hz | 与 10Hz 区分明显，60Hz 屏幕下可稳定呈现 |
| 选项 3 | 6Hz | 与前两者拉开间隔，60Hz 屏幕下可稳定呈现 |

### 4.2 第二阶段：刺激参数配置化

实现内容：

1. 在 `data/choice_trees/default_choice_tree.json` 的 choice 对象中支持可选字段：

```json
{
  "choice_id": "root.c1",
  "choice_text": "你吃过饭了吗",
  "child_node_id": "capability",
  "ssvep": {
    "frequency": 10,
    "phase": 0,
    "color": "#ffffff"
  }
}
```

2. 在 `choice/orchestrator.py` 的 `_build_choices()` 中透传 `ssvep` 元数据。
3. 前端优先使用后端返回的 `choice.ssvep.frequency`，没有配置时使用默认频率表。

### 4.3 第三阶段：BCI 识别结果接入

实现内容：

1. 新增前端函数 `selectChoiceBySsvepTarget(targetIndex)`。
2. 支持外部识别模块通过 WebSocket、HTTP 轮询或浏览器事件传入目标编号。
3. 收到目标编号后高亮锁定目标，并调用现有 `requestChoiceSelect(choiceId)`。
4. 增加防误触发机制，例如连续识别同一目标 500-1000ms 后才触发选择。

## 5. 技术方案

### 5.1 前端结构调整

计划修改 `web/dashboard.html`：

1. 在 `choice-panel` 中加入 SSVEP 控制栏。
2. 增加 CSS 类：
   - `.ssvep-enabled`
   - `.ssvep-choice`
   - `.ssvep-frequency-badge`
   - `.ssvep-locked`
3. 保持 `.choice-btn` 的基础交互语义，避免影响现有点击选择。

计划修改 `web/client.js`：

1. 增加 `ssvepState`：

```javascript
var ssvepState = {
    enabled: false,
    rafId: null,
    startTime: 0,
    frequencies: [10, 7.5, 6],
    targets: []
};
```

2. 在 `renderChoiceState(payload)` 中记录每个选项的 DOM、choiceId、frequency。
3. 增加：
   - `startSsvepFlicker()`
   - `stopSsvepFlicker()`
   - `updateSsvepTargets()`
   - `selectChoiceBySsvepTarget(targetIndex)`
4. 在 `stop()`、`requestChoiceReset()`、`renderChoiceState()` 中同步处理 SSVEP 状态。

### 5.2 闪烁实现方式

推荐使用 `requestAnimationFrame` 按时间计算亮度：

```javascript
var value = Math.sin(2 * Math.PI * frequency * elapsedSeconds + phase);
var alpha = value > 0 ? 1 : 0.25;
```

这样可以：

- 用同一帧循环管理多个频率。
- 根据真实时间计算相位，减少定时器漂移。
- 后续容易增加正弦亮度、方波亮度、相位编码等实验模式。

第一阶段建议采用方波式亮暗变化，视觉更明显；如需要降低刺激强度，可改成正弦透明度变化。

### 5.3 频率策略

默认频率按当前三选一设计：

- 选项 1：10Hz
- 选项 2：7.5Hz
- 选项 3：6Hz

如果后续支持 4 个以上选项，可扩展为：

- 6Hz
- 7.5Hz
- 8.57Hz
- 10Hz
- 12Hz

注意：屏幕刷新率会影响实际刺激稳定性。建议实验电脑固定使用 60Hz 或 120Hz 刷新率，并在界面中显示当前检测到的估算刷新率或实验配置说明。

## 6. 安全与可用性要求

1. SSVEP 模式必须默认关闭，由用户主动开启。
2. 开启前应在界面中给出简短提示：闪烁视觉刺激可能引起不适，感到眩晕或眼部疲劳应立即关闭。
3. 提供明显的关闭开关。
4. 连接停止、页面切换、重新选角时必须停止动画循环。
5. 移动端或低刷新率设备上允许自动降级为普通选项按钮。
6. 保留鼠标点击和键盘选择，保证非 BCI 场景可用。

## 7. 数据与接口设计

### 7.1 前端内部数据

```javascript
{
    index: 0,
    choiceId: "root.c1",
    frequency: 10,
    phase: 0,
    element: buttonElement
}
```

### 7.2 后端 choice tree 扩展

可选扩展字段：

```json
"ssvep": {
  "frequency": 10,
  "phase": 0,
  "mode": "square",
  "duty_cycle": 0.5
}
```

### 7.3 BCI 识别输入预留

建议预留前端统一入口：

```javascript
window.LiveTalkingSSVEP = {
    selectTarget: selectChoiceBySsvepTarget,
    enable: enableSsvepMode,
    disable: disableSsvepMode
};
```

后续 Python BCI 模块或独立采集程序可以通过 WebSocket 将识别结果传给浏览器，再调用该入口。

## 8. 实施步骤

### 阶段一：计划与原型

1. 完成项目计划书。
2. 在前端加入 SSVEP 开关和基础样式。
3. 在选项渲染时绑定默认频率。
4. 实现 `requestAnimationFrame` 闪烁循环。
5. 手动验证点击选择、重置、停止连接均正常。

### 阶段二：配置化与联调

1. 扩展 choice tree 的 `ssvep` 字段。
2. 后端 `_build_choices()` 透传 SSVEP 元数据。
3. 前端优先使用后端配置频率。
4. 增加频率标签、锁定态、高亮态。
5. 验证多轮选项切换后频率仍然正确。

### 阶段三：BCI 接入

1. 设计 BCI 识别结果格式。
2. 建立 WebSocket 或 HTTP 接入通道。
3. 实现连续识别确认机制。
4. 识别目标后自动触发 `/choice/select`。
5. 记录识别目标、置信度、触发时间、实际选择结果，便于实验分析。

## 9. 验收标准

1. 打开页面后，普通选项对话功能不受影响。
2. 开启 SSVEP 后，当前三个选项以不同频率闪烁。
3. 每次 `/choice/init` 或 `/choice/select` 返回新选项后，闪烁目标自动更新。
4. 点击选择、重新开始、停止连接不会留下异常动画循环。
5. 控制台无明显 JavaScript 报错。
6. 后续外部模块可以通过统一入口按目标编号触发选择。

## 10. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 浏览器动画帧率不稳定 | 实际频率偏移 | 使用 `requestAnimationFrame` 按真实时间计算，并在实验设备固定刷新率 |
| 闪烁导致用户不适 | 安全风险 | 默认关闭、提供提示、提供一键关闭 |
| 选项文本过长影响刺激面积 | 识别效果下降 | 固定按钮尺寸，增加亮度块或边框闪烁区域 |
| 多轮切换后动画状态残留 | 误刺激或性能问题 | 每次重新渲染前停止旧动画并重建 targets |
| BCI 识别误触发 | 错误选择 | 增加确认窗口、置信度阈值和锁定态 |

## 12. 结论

本项目建议以“前端 SSVEP 刺激原型优先”的方式推进。第一阶段不改变现有后端数据结构，只在当前三选一按钮上增加可开关的不同频率闪烁，快速验证视觉刺激和交互流程；第二阶段再将频率参数配置化；第三阶段接入真实 BCI 识别结果，实现从注视目标到自动选择的闭环。
